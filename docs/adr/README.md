# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) for the
ChatGPT Library Archiver project.

## What is an ADR?

An ADR captures a significant architectural or design decision along with its
context and consequences. We follow the
[MADR 4.0.0](https://github.com/adr/madr/blob/4.0.0/template/adr-template.md)
template.

## Naming Convention

Each ADR is a Markdown file named with a four-digit sequence number and a
kebab-case title:

```text
NNNN-title-with-dashes.md
```

For example: `0001-use-requests-for-http-client.md`.

Check this directory for the highest existing number and use the next
sequential value when creating a new record.

## Status Lifecycle

| Status | Meaning |
| ------ | ------- |
| **Proposed** | Under discussion / in review |
| **Accepted** | Merged to the main branch |
| **Deprecated** | No longer recommended but not yet replaced |
| **Superseded by [ADR-XXXX]** | Replaced by a newer decision |
