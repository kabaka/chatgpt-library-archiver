## Executive Summary

The gallery consists of **two distinct implementations**:

| | Modern (`index.html`) | Legacy (`page_*.html`) |
|---|---|---|
| **Rendering** | Client-side via `metadata.json` | Server-rendered static HTML |
| **Features** | Search (boolean logic), date filtering, size selector, lightbox, dark mode, lazy loading | Title search only, size selector, dark mode |
| **Thumbnails** | Uses `thumbs/{small,medium,large}/` | Uses full-size images for everything |
| **Pagination** | None (loads all 1 169 items) | Hardcoded multi-page split |
| **Generated by** | Current `gallery.py` | Unknown; no code generates these |

The modern `index.html` is significantly better but has **critical accessibility gaps** (no viewport meta tag, no focus trap in lightbox, XSS via `innerHTML`), **no pagination** for 1 169 items, and **missing semantic HTML**. The legacy pages are obsolete and should be removed.

### Severity Guide

| Rating | Meaning |
|---|---|
| 🔴 **Critical** | Blocks core usability or violates WCAG A |
| 🟠 **High** | Significantly degrades experience or violates WCAG AA |
| 🟡 **Medium** | Noticeable quality issue, should fix |
| 🟢 **Low** | Minor polish or nice-to-have |

## 1. Responsive Design

### What works well

- **CSS Grid with `auto-fill`/`minmax`** (`gallery_index.html:44`) — the grid naturally reflows across sizes without explicit column counts. This is the recommended pattern per the gallery skill.
- **Three responsive breakpoints** at `960px` and `680px` (`gallery_index.html:231–296`) — the header toolbar restructures progressively, hiding the title on small screens and reflowing controls.
- **Size classes** (small 150px, medium 250px, large 400px, full) let users tune density.
- **Full-size mode** uses `flex-direction: column` for a vertical stream layout.

### Issues

| # | Severity | Issue | Location |
|---|---|---|---|
| R-1 | 🔴 Critical | **Missing `<meta name="viewport">` tag.** Without it, mobile browsers render at ~980px width and everything is tiny. The gallery is effectively unusable on mobile. | `gallery_index.html:1–6` |
| R-2 | 🟡 Medium | **Date inputs overflow on narrow screens.** The `min-width: 0` fix is applied but date inputs with native pickers can still be clipped below ~360px. | `gallery_index.html:280–286` |
| R-3 | 🟡 Medium | **Gallery title hidden on mobile but no alternative branding.** `h1` is `display: none` at ≤680px with no reduced heading, so users lose context of what page they're on. | `gallery_index.html:252` |
| R-4 | 🟢 Low | **No max-width constraint on desktop.** On ultra-wide monitors (>2560px), the grid stretches very wide with large gaps between cards. A `max-width: 2000px` on `.layout` would improve readability. | `gallery_index.html:37–40` |

## 2. Accessibility (WCAG 2.1)

### What works well

- `<html lang="en">` is set correctly.
- Search input has `aria-describedby="searchHelp"`.
- Date inputs have `aria-label` attributes.
- GitHub icon has `aria-hidden="true"` on the SVG and a `.sr-only` label.
- Reset button has a `.sr-only` span for screen readers.
- `.sr-only` class is correctly implemented (clip pattern).

### Issues

| # | Severity | Issue | WCAG | Location |
|---|---|---|---|---|
| A-1 | 🔴 Critical | **No skip-to-content link.** Keyboard users must tab through all toolbar controls to reach images. The skill checklist requires this. | 2.4.1 (A) | `gallery_index.html:327` (after `<body>`) |
| A-2 | 🔴 Critical | **Lightbox has no focus trap.** When the viewer opens, focus stays wherever it was. Tab key moves to elements behind the overlay. Screen reader users have no indication the viewer opened. | 2.4.3 (A), 2.1.1 (A) | `gallery_index.html:660–698` |
| A-3 | 🔴 Critical | **Lightbox viewer `<div id="viewer">` has no ARIA role or label.** Should be `role="dialog"` with `aria-modal="true"` and `aria-label`. | 4.1.2 (A) | `gallery_index.html:316–322` |
| A-4 | 🟠 High | **Dark mode toggle is a `<div>` not a `<button>`.** Not focusable by default, no keyboard activation. Uses `onclick` handler only.** Same for search help icon (uses `<span>` with `tabindex="0"` but no `role="button"` or keyboard handler). | 2.1.1 (A), 4.1.2 (A) | `gallery_index.html:365, 343–348` |
| A-5 | 🟠 High | **Image cards lack focus indicators.** The `<a class="thumb">` elements receive focus via Tab but have no visible `:focus` or `:focus-visible` styling. | 2.4.7 (AA) | `gallery_index.html:78–82` |
| A-6 | 🟠 High | **No visible focus indicator on controls.** The `.toggle`, `.size-select`, date inputs, and reset button lack `:focus-visible` styles. Browser defaults may or may not be visible depending on the theme. | 2.4.7 (AA) | CSS section |
| A-7 | 🟠 High | **Metadata overlay only appears on hover (`:hover .meta`).** Keyboard-focused cards don't show metadata. There is no `:focus-within` trigger. | 1.3.1 (A) | `gallery_index.html:100` |
| A-8 | 🟡 Medium | **No semantic landmark elements.** Gallery uses `<div class="layout">` not `<main>`. No `<nav>` for controls. The skill checklist requires `<article>`, `<figure>`, `<figcaption>`, `<nav>`. Only `<header>` is used correctly. | 1.3.1 (A) | `gallery_index.html:327–376` |

## 3. Performance

### What works well

- **Intersection Observer lazy loading** (`gallery_index.html:407–415`) — images use `data-src` and are loaded via `IntersectionObserver` with a 200px root margin. Combined with `loading="lazy"` as a fallback.
- **Thumbnail size switching** — when changing grid size, thumbnails are swapped to the appropriately-sized version, avoiding loading full-size images in grid view.
- **`cache: 'no-store'` on metadata fetch** ensures fresh data on reload.

### Issues

| # | Severity | Issue | Location |
|---|---|---|---|
| P-1 | 🟠 High | **All 1 169 DOM nodes created on load with no pagination.** Every image card is appended to the DOM at startup. For 1 169 items, this creates ~4 700+ DOM elements (card + link + img + meta div each). On low-end devices, initial render will be slow and scrolling may jank. The skill mentions "client-side pagination" but none is implemented. | `gallery_index.html:416–458` |
| P-2 | 🟠 High | **2.3 MB `metadata.json` loaded in one fetch.** Each item has ~30–40 tags with long descriptions. For 1 169 items, this is a significant initial payload. No streaming, no pagination, no progressive loading. | `gallery_index.html:405` |
| P-3 | 🟡 Medium | **`filterGallery()` is not debounced.** The search input uses `oninput="filterGallery()"` which fires on every keystroke. The function iterates all 1 169 cards and runs a boolean parser. On slower devices this could cause input lag. Skill guidelines recommend 200–300ms debounce. | `gallery_index.html:340, 522–549` |
| P-4 | 🟡 Medium | **Cards built one-by-one with `gallery.appendChild(card)` in a loop** instead of batching via `DocumentFragment`. Each append triggers a potential reflow. | `gallery_index.html:458` |
| P-5 | 🟢 Low | **`sessionStorage` checked with `typeof` guard but called synchronously on every filter/size change.** Minor but could be wrapped in a try/catch for private browsing safety. | `gallery_index.html:527–530` |

## 4. Dark Mode

### What works well

- **System preference detection** via `@media (prefers-color-scheme: dark)` with the clever `body:not(.light)` selector — respects OS preference but allows manual override.
- **`sessionStorage` persistence** remembers theme choice across page navigations within the session.
- **CSS custom properties** (`--bg`, `--text`, `--meta-color`, `--control-bg`) make theming clean and maintainable. Matches skill patterns exactly.
- **Transition on background/color** for smooth theme switching.

### Issues

| # | Severity | Issue | Location |
|---|---|---|---|
| D-1 | 🟡 Medium | **Metadata overlay uses hardcoded `rgba(0,0,0,0.6)` background and `#fff` text** regardless of theme. Works in both themes visually, but in dark mode the black overlay on a dark card creates a low-contrast edge. Not a WCAG fail but aesthetically inconsistent. | `gallery_index.html:91–99` |
| D-2 | 🟡 Medium | **Lightbox overlay uses hardcoded `rgba(0,0,0,0.8)`** and `color: white` — doesn't use CSS variables. Fine visually but diverges from the theming architecture. | `gallery_index.html:300–314` |
| D-3 | 🟢 Low | **Toggle is text-based ("Toggle Dark Mode") with no icon.** A sun/moon icon would be more compact and recognizable, especially on mobile where the toggle is hidden at ≤680px (see `gallery_index.html:294`). | `gallery_index.html:365` |
| D-4 | 🟢 Low | **Dark mode toggle hidden on mobile** (`.controls .toggle { display: none }` at ≤680px). Mobile users cannot switch themes at all. | `gallery_index.html:294` |

## 5. Navigation & Filtering

### What works well

- **Boolean search parser** (`tokenize()` / `makeSearchFn()`) supports AND, OR, NOT, and parentheses. Searches both titles and tags. This is a genuinely powerful feature for a static gallery.
- **Date range filtering** with native date inputs.
- **Combined AND logic** between text search and date range.
- **Filter state persistence** via `sessionStorage` — filters survive page reload.
- **Reset button** clears all filters.

### Issues

| # | Severity | Issue | Location |
|---|---|---|---|
| N-1 | 🟠 High | **No pagination.** All 1 169 items render at once. Users must scroll through the entire collection. The skill documentation mentions pagination but it's not implemented in the modern gallery. | — |
| N-2 | 🟠 High | **No sort options.** Items are sorted by `created_at` descending (set in Python) but users cannot sort by title, date ascending, or tags. | `gallery.py:32–35` |
| N-3 | 🟡 Medium | **No result count shown.** After filtering, no indicator tells users how many images matched. | — |
| N-4 | 🟡 Medium | **Search help icon shows a tooltip on hover only.** No click-to-expand help text. The boolean search syntax (AND/OR/NOT/parentheses) is powerful but undiscoverable. | `gallery_index.html:343–348` |
| N-5 | 🟡 Medium | **Tag-based navigation missing.** Tags are displayed in the overlay but not clickable — clicking a tag should filter to that tag. With 1 169 richly-tagged items, tag navigation would be a high-value feature. | `gallery_index.html:438–439` |
| N-6 | 🟢 Low | **No keyboard shortcut for search focus** (e.g., `/` or `Ctrl+K`). | — |

## 6. Lightbox / Viewer

### What works well

- **Keyboard navigation**: Arrow keys for prev/next, Escape to close.
- **Touch/swipe support**: `touchstart`/`touchend` with a 50px threshold, horizontal detection (ignores vertical swipes).
- **Click-outside-to-close** behavior.
- **Ctrl/Cmd+click passes through** to open image in a new tab.
- **Visible indices tracking** — lightbox navigation skips filtered-out images.
- **Raw file link** for direct access to full-size image.

## 10. Single-File Architecture

### Strengths

- **Zero dependencies** — no CDN requests, no build tools, no frameworks. Opens instantly offline.
- **Simple deployment** — Python copies one file. No asset pipeline needed.
- **~20 KB template size** — very lightweight for the feature set.
- **Self-contained** — a user who receives the `gallery/` directory has everything needed.

### Issues

| # | Severity | Issue | Location |
|---|---|---|---|
| S-1 | 🟡 Medium | **Legacy `page_*.html` files are orphaned.** `page_1.html` (5 575 lines) and `page_2.html` (4 645 lines) are a completely different implementation with hardcoded inline images (full-size, no thumbnails), duplicate JavaScript, no lightbox, weaker filtering. They are NOT generated by the current `gallery.py` and appear to be legacy artifacts. They confuse the gallery structure and load full-size images directly, wasting bandwidth. | `gallery/page_1.html`, `gallery/page_2.html` |
| S-2 | 🟡 Medium | **CSS and JS are not minified.** At 739 lines / 20 KB uncompressed, this isn't critical, but a minified version could trim to ~12 KB. Since it's a single file, gzip/brotli compression would bring it to ~4 KB. | — |
| S-3 | 🟢 Low | **No cache-busting mechanism.** The `index.html` filename never changes. If regenerated, browser cache may serve a stale version. A query string or versioned filename would help. | `gallery.py:42` |

## 11. JavaScript Quality

### Strengths

- **Boolean search parser** is well-implemented — recursive descent with proper operator precedence (NOT > AND > OR).
- **Viewer data/index tracking** is clean — `viewerData[]` and `visibleIndices[]` correctly handle filtered sets.
- **Touch handling** distinguishes swipes from taps, respects both horizontal and vertical axes.
- **Modular functions** — `loadImages()`, `filterGallery()`, `changeSize()`, `toggleDarkMode()` are reasonably separated.

### Issues

| # | Severity | Issue | Location |
|---|---|---|---|
| J-1 | 🔴 Critical | **XSS via `innerHTML` with unsanitized metadata.** Title, tags, filenames, and URLs from `metadata.json` are concatenated into HTML strings and injected via `card.innerHTML`. A title like `<img src=x onerror=alert(1)>` executes JavaScript. This was also identified in the security audit (H-1). | `gallery_index.html:441–450` |
| J-2 | 🟡 Medium | **Global scope pollution.** All functions (`loadImages`, `filterGallery`, `toggleDarkMode`, etc.) and variables (`viewerData`, `visibleIndices`, `currentIndex`) are in the global scope. An IIFE or module pattern would be cleaner. | Throughout `<script>` |
| J-3 | 🟡 Medium | **Inline event handlers.** `oninput="filterGallery()"`, `onchange="filterGallery()"`, `onclick="toggleDarkMode()"` — these require global function names and are harder to maintain. Should use `addEventListener`. | `gallery_index.html:340, 350, 351, 365` |
| J-4 | 🟡 Medium | **`sessionStorage` calls not wrapped in try/catch.** In Safari private browsing (older versions) and some security-restricted contexts, `sessionStorage` throws. The `typeof` guard doesn't catch quota/access exceptions. | Multiple locations |
| J-5 | 🟢 Low | **`loadImages().then(filterGallery)` — no `.catch()`.** If metadata loading fails, the error is silently swallowed. | `gallery_index.html:738` |
| J-6 | 🟢 Low | **Magic numbers.** Swipe threshold `50`, root margin `'200px 0px'`, tag limit `5` — these should be named constants for readability. | Various |

## Prioritized Recommendations

### P0 — Critical (fix immediately)

1. **Add `<meta name="viewport" content="width=device-width, initial-scale=1">`** to `<head>`. Without this, the gallery is broken on every mobile device. [R-1, B-1]

2. **Sanitize all metadata before HTML injection.** Add an `escapeHtml()` function and apply it to every value interpolated into `innerHTML`, or refactor to use `createElement()`/`textContent`. [J-1] — Already flagged in security audit H-1.

3. **Add focus management to the lightbox.** On open: move focus to viewer, set `role="dialog"`, `aria-modal="true"`, trap Tab within. On close: return focus to the card that triggered it. [A-2, A-3, L-1, L-2]

4. **Add a skip-to-content link** as the first focusable element in `<body>`. [A-1]

### P1 — High (fix soon)

5. **Implement client-side pagination** (e.g., 100 items per page, infinite scroll or page buttons). For 1 169 items, this dramatically improves initial load time and scroll performance. [P-1, N-1]

6. **Make all controls keyboard-accessible.** Change the dark mode toggle from `<div>` to `<button>`. Add `role="button"` and `keydown` handler to the help icon. [A-4]
